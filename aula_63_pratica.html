💻 Aula 63 — Bibliotecas de limpeza: introdução — prática de laboratório

📅 02/06/2026 ⏱ 50 minutos 🎯 Escola que Queremos 📚 Coleta e Limpeza de Dados I
Laboratório de informática

🎯 O que vamos aprender?

Nesta aula prática, a turma vai observar registros com pequenos problemas de preenchimento e aplicar uma rotina simples de organização e correção. O foco é perceber, na prática, como a limpeza dos dados melhora a qualidade da base do projeto.

💡 Aplicação no projeto: compreender, na prática, que existem rotinas que ajudam a preparar e limpar os dados do projeto “Escola que Queremos”.

📘 Conteúdo principal

Conceito central: lógica prática de limpeza dos dados

A limpeza dos dados ajuda a corrigir erros simples, padronizar campos e organizar melhor os registros. Nesta prática, a turma não vai trabalhar com programação avançada: vai aplicar uma sequência guiada de conferência, correção e padronização em uma planilha.

Roteiro da aula (50 min)

TempoO que vamos fazer

🧪 Atividade guiada de laboratório

Abra a planilha indicada pelo professor e siga o passo a passo abaixo.

  1. Observe a planilha: leia as colunas e veja quais informações aparecem em cada registro.
  2. Encontre problemas: procure células vazias, palavras escritas de formas diferentes, respostas repetidas e formatos diferentes para a mesma informação.
  3. Padronize os campos: deixe iguais os nomes de turmas, grupos, datas, espaços ou respostas fechadas, mantendo um único padrão.
  4. Confira a origem: verifique se cada linha continua identificada com grupo, público ou fonte de onde veio o dado.
  5. Salve a versão corrigida: registre a planilha final já organizada.
O que observarExemplos de problema
Campos vazioslinha sem turma, sem grupo ou sem resposta
Falta de padrão"1A", "1º A" e "Primeiro A" para a mesma turma

📦 Microentrega da aula: planilha curta corrigida, com os principais problemas encontrados e as correções realizadas.

📝 Resumo para o Caderno — Aula 63 Prática

  1. Limpar dados significa corrigir, organizar e preparar os registros para uso no projeto.
  2. Na prática, isso inclui localizar erros simples, campos vazios, duplicidades e falta de padrão.
  3. Padronizar ajuda a deixar a base mais clara e confiável.
  4. Corrigir não é inventar informação nova, e sim ajustar o que já foi coletado.
  5. A origem do dado precisa continuar identificada mesmo depois da limpeza.
  6. Uma base bem organizada ajuda as próximas etapas do projeto.

✅ Fechamento da prática

Ao final da atividade, a turma deve perceber que a limpeza dos dados é uma etapa técnica importante para tornar a base do projeto mais confiável, organizada e pronta para uso.

Ideia principal: entender a lógica de limpeza ajuda a construir uma base mais confiável, rastreável e pronta para o projeto.
