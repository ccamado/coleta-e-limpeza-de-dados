💻 Aula 64 — Explicação do uso de bibliotecas de limpeza — prática de laboratório

📅 02/06/2026 ⏱ 50 minutos 🎯 Escola que Queremos 📚 Coleta e Limpeza de Dados I
Laboratório de informática

🎯 O que vamos aprender?

Nesta aula prática, a turma vai comparar registros diferentes e aplicar uma regra comum de padronização. O objetivo é perceber como a lógica da limpeza ajuda a organizar, corrigir e deixar a base do projeto mais coerente.

💡 Aplicação no projeto: perceber, em situação prática, como a lógica de limpeza ajuda a organizar, corrigir e padronizar os registros do projeto.

📘 Conteúdo principal

Conceito central: padronização prática de registros

Quando cada grupo registra de um jeito, a base fica difícil de usar. Nesta prática, a turma vai aplicar uma sequência guiada para comparar registros, definir um padrão comum e corrigir inconsistências sem alterar o sentido original dos dados.

🧪 Atividade guiada de laboratório

Abra a planilha indicada pelo professor e siga este passo a passo.

  1. Compare os registros: observe linhas feitas por grupos diferentes e identifique onde a mesma informação aparece com formatos diferentes.
  2. Defina um padrão: escolha um único jeito de registrar turma, data, local e respostas fechadas.
  3. Corrija as inconsistências: ajuste grafias, abreviações, uso de maiúsculas e minúsculas e formatos diferentes para a mesma informação.
  4. Revise duplicidades: veja se há linhas repetidas ou quase iguais.
  5. Documente o que foi feito: registre quais regras de padronização foram usadas pelo grupo.
CampoExemplo de padronização
Turmausar sempre o mesmo formato para todas as turmas
Dataregistrar todas as datas no mesmo padrão
Espaçoescrever o nome do local sempre da mesma forma
Resposta fechadausar um padrão único para sim/não ou categorias equivalentes
📦 Microentrega da aula: tabela padronizada com uma legenda curta das regras de limpeza usadas pelo grupo.

📝 Resumo para o Caderno — Aula 64 Prática

  1. Padronizar dados significa registrar informações do mesmo tipo sempre do mesmo jeito.
  2. Quando cada grupo registra de forma diferente, a base fica difícil de organizar.
  3. A limpeza ajuda a corrigir grafias, formatos e duplicidades.
  4. Padronizar não é mudar o sentido da informação, e sim deixá-la coerente.
  5. Registrar as regras usadas melhora a rastreabilidade da limpeza.
  6. Uma base coerente facilita o trabalho das próximas etapas do projeto.

✅ Fechamento da prática

Ao final da atividade, a turma deve reconhecer que a lógica de limpeza só faz sentido quando ajuda a manter os registros claros, coerentes e prontos para o uso coletivo no projeto.

Ideia principal: usar a lógica de limpeza ajuda a deixar os registros do projeto coerentes entre si e prontos para as próximas etapas.
